# 文档质量检查规范

完成初稿后，需按以下标准进行自查与修改：

## 1. 主题贴合度检查
- 核心主题是否明确：是否围绕"文本反思+专业结合+时代担当"的正向主线展开？
- 是否存在偏离主题的消极、厌世内容？是否所有段落都服务于核心主题？

## 2. 内容完整性检查
- 是否覆盖了topic.md中要求的所有要点？
- 两本书的文本解读与对比是否全面准确？
- 各章节内容是否完整，逻辑是否连贯？

## 3. 专业结合度检查
- 软件工程专业与两本书的结合是否自然、不生硬？
- 专业相关的思考是否符合大二学生的认知水平，无错误表述？

## 4. 字数与结构检查
- 全文是否达到3000字以上？
- 是否严格按照structure.md的提纲撰写，章节清晰，层次分明？

## 5. 语气与价值观检查
- 是否符合voice.md的规范，语言正式、真诚，无口语化/网络化表达？
- 整体基调是否积极向上，传递正确的价值观，无过度消极的表述？

## 6. 逻辑与表达检查
- 段落之间的过渡是否自然，逻辑是否连贯？
- 是否存在病句、错别字、标点错误？
- 引用文本内容是否准确，无事实性错误？

## 修改规则
- 不符合上述任意一项的内容，需进行针对性修改，修改后重新检查，直至全部达标。
