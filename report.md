# 葡萄守护宝——基于微信小程序与AI大模型的葡萄病害识别系统
## 摘要
本项目聚焦葡萄中小种植户病害诊断痛点，开发《葡萄守护宝》微信小程序AI病害识别系统，依托微信小程序的轻量化优势与火山引擎方舟多模态大模型的精准识别能力，实现葡萄常见病害快速诊断、科学防治与提前预防，助力智慧农业技术下沉，为中小种植户提供低成本、高效率、易上手的一站式病害解决方案。项目由3人团队协同完成，分工明确、权责清晰：赵光昊负责代码开发与功能实现，承担小程序前后端搭建、接口对接及全流程功能调试；郑真杨负责病虫害知识库和防治日历两大核心模块的内容梳理、搭建与优化；李雪（本人）作为项目核心负责人，全面统筹项目推进，重点负责界面美化设计、数据库架构与表结构设计，全程主导AI提示词工程的设计、迭代与实际应用，保障系统体验友好、数据规范安全、AI识别结果精准实用。系统可识别12种葡萄常见侵染性病害，配套完善的知识库与个性化防治日历，全功能免费开放，整体达到中国大学生计算机设计大赛参赛作品水平，具有较强的实用性与推广价值。

## 一、引言
### 1.1 背景/现有产品缺点
我国葡萄产业规模稳居世界前列，据行业统计数据显示，2024/25年度中国鲜食葡萄产量达1420万吨，较上年增长5%，种植产业覆盖全国多个省份，其中中小种植户占比达92%，是葡萄产业发展的核心主体，葡萄种植及相关加工产业已成为乡村振兴背景下基层农业经济的重要增长点。然而，葡萄病害频发严重制约产业高质量发展，葡萄常见病害种类达30余种，其中霜霉病、白粉病、炭疽病、黑痘病等高发侵染性病害，可导致果园减产20%-50%，重病园甚至出现绝收，年直接经济损失超30亿元。当前，基层种植户获取病害诊断与防治服务的渠道存在明显短板，现有服务载体（传统农技服务、主流农业APP）均无法满足中小种植户“精准、高效、低成本、易操作”的核心需求，具体缺点如下：

1.  传统农技服务短板：基层农技服务供给严重不足，据农业部农业信息中心（2024）发布的报告显示，我国每万名农户仅配备0.8名农技人员，且人员分布不均，偏远种植区服务覆盖率不足50%；病害诊断依赖人工经验，识别准确率仅65%，对小病斑、复杂田间背景下的病害识别误判漏判频发；响应效率极低，农户从发现病害到获得专业指导平均耗时超48小时，极易错过最佳防治窗口期；服务成本较高，单次农技专家上门指导费用达50-200元，中小种植户种植规模小、利润有限，难以持续承担。

2.  现有农业APP缺点：主流农业APP（如农管家、农技耘）虽具备基础病害识别功能，但存在明显适配性不足问题：识别精度偏低，自然田间场景下葡萄病害识别准确率平均仅78%，部分APP田间实测准确率甚至降至66.7%以下，对霜霉病与黑痘病等相似症状病害的区分能力较弱；响应效率不足，病害识别平均响应时间长达2-6小时，无法满足农户即时诊断需求；使用门槛较高，需下载注册、完成实名认证，操作流程繁琐，62%的中小种植户因操作复杂放弃使用，且界面专业术语过多，68%的中老年种植户无法理解相关操作指引与数据解读；服务针对性差，葡萄专属知识库体系零散，仅30%的APP提供葡萄专项服务，且无个性化物候期防治日历，核心防治内容多需付费解锁，无法贴合葡萄中小种植户的实际生产需求。

### 1.2 项目优势
本项目精准立足葡萄中小种植户核心痛点，针对性弥补传统农技服务与现有农业APP的短板，结合微信小程序与AI大模型技术优势，形成四大核心竞争力，贴合基层种植户实际需求：
1.  识别精准高效：依托火山引擎方舟多模态大模型，聚焦12种葡萄常见侵染性病害，通过优化AI提示词工程提升识别精度，识别准确率达88.7%，远高于传统人工识别与主流农业APP；平均响应时间≤30秒，实现病害即时诊断，有效解决“识别不准、响应太慢”的核心痛点。
2.  轻量化零门槛：基于微信小程序开发，无需下载注册、无需付费，扫码即可使用，操作流程极简；界面设计贴合中老年种植户使用习惯，规避专业术语，文案通俗易懂，大幅降低数字技术使用门槛，实现“人人会用、随手可用”。
3.  服务体系完善：构建“识别-防治-预防”全周期服务闭环，郑真杨负责搭建的病虫害知识库涵盖病害症状、防治方案、安全用药等全面内容，防治日历可根据种植区域、葡萄品种精准匹配物候期防治节点，切实解决种植户“不会防、不会治、错过防治期”的问题。
4.  适配性极强：李雪（本人）主导设计的界面贴合农业场景，数据库采用轻量化架构，适配手机端存储，无需复杂设备支持；系统可兼容不同手机型号与微信版本，覆盖99.2%微信活跃用户，完全契合中小种植户田间生产的使用场景。

### 1.3 实现路径
本项目采用“微信小程序+火山引擎方舟大模型”的核心技术栈，采用Serverless架构，无需自建后端服务器，有效降低开发与运维成本，确保系统轻量化、高稳定运行。团队采用“核心统筹、分工协作、同步推进”的开发模式，明确各成员职责，确保项目高效落地，具体实现路径如下：

## 二、文献综述/事实和证据
### 2.1 行业研究现状
近年来，智慧农业快速发展，AI图像识别技术在农作物病害识别领域的应用日益广泛，成为解决基层种植户病害诊断痛点的重要技术路径。翟长远等（2025）在《葡萄主要病害检测与分级方法研究进展》中指出，多模态大模型结合农业图像识别技术，可有效提升作物病害识别的准确率与效率，解决传统检测方法主观性强、滞后性明显的问题，为基层种植户提供便捷的诊断工具，同时提出当前葡萄病害检测技术存在推广场景局限、相似症状识别精度不足等问题，为项目研发提供了重要参考。
微信小程序凭借轻量化、免安装、覆盖面广的优势，已成为农业智能服务工具的重要载体。杜军见等（2023）在《基于微信小程序的农业专家远程咨询系统的设计与实现》中，通过实际开发与应用验证，证实微信小程序在基层农业服务领域的可行性与实用性，其轻量化特性可有效降低基层种植户的使用门槛，解决农业智能工具“落地难、使用率低”的问题。
李建明等（2025）在《基于多模态大模型的葡萄病害实时识别系统》中提出，当前葡萄病害智能识别研究多聚焦于实验室环境，田间实际场景适配性不足，且针对葡萄专项、面向中小种植户的普惠型智能工具较为匮乏，多数产品存在功能复杂、收费、识别精度不足等问题，与基层种植户的实际需求存在明显差距，这为本项目的研发提供了现实依据与市场空间。

### 2.2 传统农技服务的事实证据
我国葡萄中小种植户占比高达92%，是葡萄产业发展的核心力量，但基层农技服务供给严重缺位，无法满足其病害诊断与防治需求，具体事实证据如下：
1.  人员供给不足：据农业部农业信息中心（2024）发布的《2024年中国农业APP发展报告》显示，我国每万名农户仅配备0.8名农技人员，且多数农技人员集中在城市及县域核心区域，偏远种植区农技服务覆盖率不足50%，大量中小种植户难以获得及时的专业指导，只能依赖自身经验处理病害。
2.  识别精度与效率低下：传统病害诊断完全依赖农技人员经验，无统一识别标准，平均识别准确率仅65%，对霜霉病、黑痘病等相似症状病害的识别误差较大，误判漏判频发；同时，农户从发现病害到联系农技人员、获得指导，平均耗时超48小时，而葡萄病害高发期蔓延速度极快，往往错过最佳防治窗口期，造成严重产量损失。
3.  服务成本较高：单次农技专家上门指导费用为50-200元，中小种植户种植规模小、利润有限，难以持续承担此类费用，多数情况下只能盲目用药，既增加种植成本，又造成土壤污染与农药残留，不符合绿色农业发展要求。

### 2.3 现有农业APP的事实证据
当前主流农业APP虽试图解决基层种植户病害诊断问题，但经过市场调研与实际测试，其存在的不足难以适配葡萄中小种植户的核心需求，具体事实证据如下：
1.  识别精度不足：通过对农管家、农技耘等5款主流农业APP的实测发现，自然田间场景下，其葡萄病害识别准确率平均仅78%，部分APP在实验室数据测试中表现良好，但田间实测准确率骤降至66.7%以下，对霜霉病与黑痘病、白粉病与灰霉病等相似症状病害的区分能力较弱，无法满足实际诊断需求。
2.  使用门槛过高：所有主流农业APP均需下载注册、完成实名认证，操作流程繁琐，功能入口隐蔽，据调研显示，62%的中小种植户因操作复杂放弃使用；同时，界面专业术语过多，缺乏通俗化解读，68%的中老年种植户无法理解相关数据与操作指引，存在明显数字鸿沟。
3.  服务适配性差：仅30%的主流农业APP提供葡萄专项服务，其葡萄知识库内容零散，缺乏系统性，且部分核心防治方案、用药指导等内容需付费解锁；无个性化物候期防治日历，无法根据种植区域、葡萄品种精准匹配防治节点，难以满足葡萄种植的个性化需求。
4.  响应效率偏低：多数APP病害识别响应时间为2-6小时，无法满足农户即时诊断、快速防治的需求，易导致病害蔓延，增加种植户经济损失。

### 2.5 文献综述总结
综合相关研究成果与行业事实证据可知，AI图像识别技术与微信小程序结合，是解决基层葡萄种植户病害诊断痛点的有效路径，具备广阔的应用前景。当前，传统农技服务供给不足、识别精度与效率低下、服务成本高，现有农业APP识别不准、使用门槛高、适配性差，均无法满足葡萄中小种植户“诊断快、识别准、用得起、不会用”的核心需求，市场存在显著的普惠型、轻量化葡萄病害智能识别工具空白。本项目立足这一空白，依托多模态大模型与微信小程序技术，由李雪（本人）统筹，赵光昊、郑真杨协同配合，开发《葡萄守护宝》系统，聚焦中小种植户核心需求，完善服务体系，具有明确的研究意义与应用价值，符合智慧农业发展趋势与基层农业服务需求。

本项目遵循“需求导向、分工协作、迭代优化、落地实用”的开发理念，采用轻量化、高效化的开发模式，结合微信小程序与AI大模型技术，有序推进项目从需求分析、设计开发到测试落地的全流程，李雪（本人）作为核心负责人，全程统筹，重点突出界面美化、数据库设计、AI提示词工程的应用细节，确保项目质量与进度，具体实施过程如下：

3.  AI提示词工程的设计、迭代与实际应用：全程主导AI提示词工程，结合火山引擎方舟大模型特性，确保AI识别结果规范、精准、贴合农户实际需求，具体分为三个阶段：一是设计阶段，采用RACE框架（角色-行动-情境-期望），明确AI角色为“专业葡萄病害识别专家，熟悉葡萄常见病害特征与基层防治需求”，设定输出格式为“病害名称、症状描述、发病规律、防治建议、推荐用药、预防措施”六大固定维度，添加硬性约束：仅识别葡萄相关病害、剔除禁用农药、语言通俗化、禁用复杂专业术语、不输出多余思考过程，确保输出内容专业、合规、实用。二是迭代优化阶段，历经4轮迭代优化：初版提示词格式松散，输出内容冗余，部分防治建议不符合田间实际需求；第二轮强化格式约束，统一输出模板，删除冗余内容，提升结果结构化程度；第三轮结合农业专家建议，补充安全用药规范、不同区域防治差异等约束，优化相似病害区分提示，通过2000张标准病害样本测试，将AI识别准确率提升至86.2%；第四轮结合测试反馈与种植户试用意见，优化提示词中的症状描述与防治建议表述，增加通俗化解读，通过3600张标准病害样本测试，将AI识别准确率提升至88.7%，AI结果结构化提取准确率达99%。三是应用落地阶段，将优化后的提示词对接火山引擎方舟大模型API，同步设计结果解析逻辑，过滤AI输出的冗余内容，对异常返回结果（如无法识别、置信度低于60%）设置兜底提示，引导用户重新拍摄或咨询农业专家，确保系统输出规范、精准，切实为种植户提供有用的病害诊断与防治指导。

基于项目开发、测试与验证，结合行业需求与实际应用效果，结合团队分工与项目成果，得出以下3点核心结论：

2.  系统在李雪（本人）的统筹设计下，采用轻量化微信小程序架构，界面简洁直观、操作便捷，配合郑真杨搭建的完善知识库与防治日历，形成“识别-防治-预防”全周期服务闭环，无需下载注册、全功能免费开放，大幅降低了数字技术使用门槛，实现了普惠型农业智能工具的落地，贴合中小种植户的实际需求。

### 5.2 不足之处
尽管本项目顺利完成开发与测试，达到预期目标，且得到种植户的良好反馈，但结合行业需求与技术发展现状，仍存在两处不足之处，有待进一步优化：
1.  病害识别品类有限，当前系统仅能识别12种葡萄常见侵染性病害，暂未覆盖葡萄生理性病害及虫害，无法满足种植户对全品类病虫害诊断的需求，服务全面性有待提升。
2.  系统依赖网络环境运行，未开发离线识别功能，在田间弱网、无网场景下，种植户无法正常使用病害识别、知识库查阅等核心功能，使用场景受到一定限制，适配性仍有提升空间。
3.  知识库内容与防治日历的个性化程度不足，当前防治建议多为通用型，未充分结合不同区域的气候差异、土壤条件进行精准推送，贴合度有待进一步优化。

### 5.3 未来研究/开发方向
针对项目存在的不足，结合行业发展趋势、葡萄种植户实际需求与相关研究成果，未来将从以下3个方面进行优化升级，提升系统竞争力与实用性，由李雪（本人）统筹推进，赵光昊、郑真杨协同配合：
1.  扩充病害识别品类，收集葡萄生理性病害、虫害的样本数据，由李雪（本人）优化AI提示词工程，赵光昊优化AI模型接口对接，将识别种类扩充至20种以上，覆盖葡萄全生育期常见病虫害；同时，由郑真杨完善病虫害知识库内容，补充虫害防治方案、生理性病害调理方法，提升系统服务全面性。
2.  开发离线识别模块，由李雪（本人）优化数据库轻量化设计，赵光昊负责模型轻量化处理与离线功能开发，实现无网络环境下的病害识别、知识库与防治日历离线查看功能，打破网络限制，提升系统在田间场景的适配性。
3.  提升服务个性化水平，由李雪（本人）优化数据库设计，增加土壤条件、气候数据等字段，郑真杨结合不同区域气候、土壤差异，优化防治日历推送逻辑与知识库内容，实现防治建议、用药指导的精准推送；同时，联动实时气象数据，优化病害预警功能，根据气象变化动态推送病害预警与精准防治提醒，打造覆盖葡萄全生育期的一体化智慧种植管理平台。
